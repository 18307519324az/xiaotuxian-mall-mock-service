# 启动说明

## 环境要求

- JDK 17
- Maven 3.8+

## 构建

```bash
mvn clean package -DskipTests
```

## 运行

```bash
java -jar target/xtx-mock-service-1.0.0.jar
```

服务默认监听：`http://localhost:8099`

## 前端代理

Vue 前端开发服务器可以将 `/api` 代理到 `http://localhost:8099`。

## 适合的启动用途

该服务最适合以下场景：

- 本地页面联调
- README 截图制作
- 公开演示时快速跑通页面流程
- 不方便启动完整真实服务链路时的替代环境

## 常见启动检查

启动后建议先检查：

- `8099` 端口是否已监听
- 首页分类接口是否可访问
- 购物车与结算接口是否返回 `result`
- 前端代理后页面是否能正常渲染

这些检查能帮助快速判断当前环境是否适合继续做截图或联调。

## 与真实服务的切换说明

当需要演示 Redis Lua、RabbitMQ 或真实订单状态流转时，应切回主仓真实微服务模式。Mock Service 更适合支撑页面展示，不负责解释真实中间件链路。

## 端口与网络说明

默认情况下，Mock Service 监听 `8099` 端口。前端开发代理通常会把 `/api/**` 转发到这个端口，因此只要 `8099` 正常可达，大多数页面就能跑通基础展示链路。

## 常见问题排查

如果页面出现空白、字段缺失或截图乱码，建议依次排查：

1. 服务是否正常启动
2. JSON 数据是否存在坏字符
3. 前端代理是否仍指向 `8099`
4. 运行时 `data/` 目录是否残留旧状态

这也是 README 截图制作前建议先做的检查。
